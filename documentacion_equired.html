1. Introducción y Visión General

EquiRed es una plataforma web social diseñada para contribuir al Objetivo de Desarrollo Sostenible 10 (ODS 10): la reducción de las desigualdades. La plataforma conecta a personas vulnerables con oportunidades de empleo, asesoría profesional (psicológica y jurídica) y apoyo solidario a través de donaciones.

Características principales

  • Feed Social: Los usuarios pueden compartir historias de superación, dar "me gusta" y comentar publicaciones.
  • Bolsa de Empleo Inclusivo: Empresas publican ofertas laborales y los usuarios pueden postularse.
  • Sistema de Donaciones: Los donantes contribuyen económicamente a programas de inclusión.
  • Asesorías Profesionales: Psicólogos y abogados ofrecen orientación a personas que han sufrido discriminación.
  • Sistema de Autenticación: Registro e inicio de sesión con diferentes roles de usuario.

Roles de usuario

RolDescripciónPermisos especiales
beneficiarioPersona que busca oportunidades y apoyoPostularse a empleos, solicitar asesorías
empresaOrganización que ofrece empleo inclusivoPublicar ofertas de empleo
profesionalPsicólogo/a o abogado/aRecibir y gestionar solicitudes de asesoría
donantePersona que contribuye económicamenteRealizar donaciones y ver historial

5. API — Configuración (config.php)

api/config.php

Archivo central de configuración que todos los demás archivos PHP importan. Contiene la conexión a la base de datos y funciones auxiliares reutilizables.

5.1. Headers HTTP (CORS)

header('Content-Type: application/json; charset=utf-8');
header('Access-Control-Allow-Origin: *');
header('Access-Control-Allow-Methods: GET, POST, PUT, DELETE, OPTIONS');
header('Access-Control-Allow-Headers: Content-Type');

Estos encabezados configuran:

  • Content-Type: Indica que todas las respuestas serán en formato JSON.
  • CORS (Cross-Origin Resource Sharing): Permite que el frontend acceda a la API desde cualquier origen (*). Esto es necesario cuando el frontend y la API se sirven desde diferentes orígenes.
  • Preflight: Las peticiones OPTIONS (preflight) se responden con código 200 inmediatamente.

5.2. Conexión a la Base de Datos

define('DB_HOST', 'localhost');
define('DB_NAME', 'equired_db');
define('DB_USER', 'root');
define('DB_PASS', '');
define('DB_CHARSET', 'utf8mb4');

function getConnection() {
    $dsn = "mysql:host=" . DB_HOST . ";dbname=" . DB_NAME . ";charset=" . DB_CHARSET;
    $options = [
        PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
        PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
        PDO::ATTR_EMULATE_PREPARES => false,
    ];
    return new PDO($dsn, DB_USER, DB_PASS, $options);
}

Usa PDO (PHP Data Objects) para una conexión segura a MySQL. Las opciones configuran:

  • ERRMODE_EXCEPTION: Los errores de BD lanzan excepciones (fácil de depurar).
  • FETCH_ASSOC: Los resultados se devuelven como arrays asociativos (clave → valor).
  • EMULATE_PREPARES = false: Usa prepared statements nativos del servidor (mayor seguridad contra inyección SQL).

5.3. Funciones auxiliares

sendResponse($data, $code)

Envía una respuesta JSON exitosa con el formato {"success": true, "data": ...}.

sendError($message, $code)

Envía una respuesta JSON de error con el formato {"success": false, "error": "..."}.

getBody()

Lee el cuerpo de la petición HTTP (que viene en formato JSON desde el frontend) y lo convierte a un array PHP.

requireAuth()

Verifica que el usuario tenga una sesión activa. Si no la tiene, devuelve un error 401. Si la tiene, retorna el user_id.

getCurrentUser()

Retorna los datos del usuario actual almacenados en la sesión PHP, o null si no hay sesión.

6. API — Autenticación (auth.php)

api/auth.php

Maneja el registro de usuarios, inicio de sesión, cierre de sesión y verificación de sesión activa.

6.1. Enrutamiento

El archivo recibe un parámetro action por la URL y ejecuta la función correspondiente:

$action = $_GET['action'] ?? '';
switch ($action) {
    case 'register': register(); break;
    case 'login':    login();    break;
    case 'logout':   logout();   break;
    case 'session':  checkSession(); break;
}

6.2. Función register()

Proceso de registro:

  1. Verifica que el método HTTP sea POST.
  2. Extrae los datos del body JSON: nombre, email, password, tipo_usuario.
  3. Validaciones: campos no vacíos, email válido, contraseña mínimo 6 caracteres, contraseñas coinciden, tipo de usuario válido.
  4. Verifica que el email no esté ya registrado en la base de datos.
  5. Encripta la contraseña con password_hash($password, PASSWORD_BCRYPT).
  6. Inserta el nuevo usuario en la BD.
  7. Inicia sesión automáticamente guardando los datos en $_SESSION.
  8. Retorna los datos del usuario creado.
🔐 Seguridad: La contraseña NUNCA se almacena en texto plano. Se usa bcrypt, un algoritmo de hash unidireccional seguro. Esto significa que aunque alguien acceda a la base de datos, no podrá ver las contraseñas reales.

6.3. Función login()

Proceso de inicio de sesión:

  1. Recibe email y password del body.
  2. Busca al usuario por email en la BD.
  3. Compara la contraseña con password_verify() (compara contra el hash almacenado).
  4. Si es correcto, crea la sesión PHP con los datos del usuario.
  5. Devuelve los datos del usuario.

6.4. Función logout()

Destruye la sesión PHP con session_destroy(), lo que cierra la sesión del usuario.

6.5. Función checkSession()

Verifica si hay una sesión activa. El frontend llama a esta función al cargar cada página para saber si el usuario está logueado.

7. API — Publicaciones (publicaciones.php)

api/publicaciones.php

Gestiona el feed social: listar, crear, dar like, comentar y eliminar publicaciones.

7.1. Enrutamiento

Usa el método HTTP y el parámetro action para determinar la acción:

MétodoActionFunción
GET—getPublicaciones() — Listar publicaciones
POSTcreatecreatePublicacion() — Crear nueva
POSTliketoggleLike() — Dar/quitar me gusta
POSTcommentaddComment() — Agregar comentario
DELETE—deletePublicacion() — Eliminar

7.2. getPublicaciones()

Obtiene las publicaciones del feed con paginación (10 por página). Para cada publicación retorna:

  • Datos del autor (nombre, avatar, tipo).
  • Conteo total de likes y comentarios (usando subconsultas SQL).
  • Si el usuario actual ya dio "me gusta" (user_liked).
  • Los últimos 5 comentarios de la publicación.

7.3. toggleLike()

Funciona como un interruptor: si el usuario ya dio like, lo quita (DELETE). Si no lo ha dado, lo agrega (INSERT). Retorna el estado actual y el total de likes.

7.4. addComment()

Inserta un nuevo comentario en la tabla comentarios y devuelve el comentario creado con los datos del autor.

7.5. deletePublicacion()

Solo permite eliminar publicaciones propias (la cláusula WHERE usuario_id = ? asegura esto). Si no se encuentra la publicación o no es del usuario, retorna error 404.

    checkSession();    // Verifica si hay sesión activa
    initNavbar();      // Configura la barra de navegación
    initAnimations();  // Activa las animaciones de entrada
});

El evento DOMContentLoaded se ejecuta cuando el HTML ha sido completamente cargado, antes de que imágenes y recursos externos terminen.

12.3. checkSession()

Hace una petición GET a api/auth.php?action=session para verificar si el usuario tiene una sesión activa. Si la sesión existe, almacena los datos en currentUser y actualiza la barra de navegación.

12.4. initNavbar()

Tres funcionalidades:

  • Efecto scroll: Agrega la clase scrolled al navbar cuando el usuario baja más de 10px.
  • Menú móvil: El botón hamburguesa (☰) alterna la visibilidad del menú en dispositivos móviles.
  • Link activo: Detecta la página actual y resalta el enlace correspondiente en la navegación.

12.5. updateNavbarAuth(isLoggedIn)

Modifica dinámicamente los botones del navbar:

  • Logueado: Muestra avatar con iniciales, nombre del usuario y botón "Salir".
  • No logueado: Muestra botones "Ingresar" y "Registrarse".

12.6. Funciones utilitarias

getInitials(name)

Extrae las iniciales de un nombre. Ejemplo: "María González" → "MG".

timeAgo(dateStr)

Convierte una fecha en un texto relativo: "Hace un momento", "Hace 5 min", "Hace 3 horas", "Hace 2 días", etc.

formatCurrency(amount)

Formatea un número como moneda usando Intl.NumberFormat. Ejemplo: 1500 → "US$1,500.00".

12.7. showToast(message, type)

Sistema de notificaciones temporales (toasts) que aparecen en la esquina de la pantalla. Tipos: success (✅), error (❌), warning (⚠️), info (ℹ️). Se auto-eliminan después de 4 segundos.

12.8. apiFetch(url, options)

Wrapper alrededor de fetch() que:

  • Agrega automáticamente el header Content-Type: application/json.
  • Parsea la respuesta JSON.
  • Captura errores de red y devuelve un formato consistente.

13. JavaScript — Landing Page (landing.js)

js/landing.js

Solo se carga en index.html. Contiene la animación de los contadores estadísticos.

13.1. initCounters()

Usa el Intersection Observer API para detectar cuándo los contadores aparecen en la pantalla. Cuando el usuario hace scroll hasta la sección de estadísticas, los números se animan de 0 al valor objetivo.

13.2. animateCounter(element, target)

Anima un número de 0 al valor objetivo en 2 segundos usando requestAnimationFrame. Usa una función de easing (ease-out: 1 - (1 - t)³) para que la animación sea fluida y desacelere al final. Al terminar, agrega un "+" al número.

14. JavaScript — Autenticación (auth.js)

js/auth.js

Se carga en login.html y registro.html. Maneja los formularios de autenticación.

14.1. handleLogin(e)

Proceso cuando el usuario envía el formulario de login:

  1. e.preventDefault() — Evita que la página se recargue (comportamiento por defecto del form).
  2. Extrae email y password de los inputs.
  3. Validación básica: ambos campos deben tener valor.
  4. Deshabilita el botón y cambia su texto a "Ingresando...".
  5. Envía petición POST a api/auth.php?action=login.
  6. Si es exitoso: muestra toast de bienvenida y redirige a principal.html después de 800ms.
  7. Si falla: muestra el error y re-habilita el botón.

14.2. handleRegister(e)

Similar al login pero con más validaciones:

  • Campos no vacíos.
  • Email contiene "@".
  • Contraseña mínimo 6 caracteres.
  • Contraseñas coinciden.

Envía nombre, email, password, confirm_password y tipo_usuario al servidor.

15. JavaScript — Feed Social (principal.js)

js/principal.js

Se carga en principal.html. Gestiona todo el feed social: publicaciones, likes y comentarios.

15.1. Variables de estado

let currentPage = 1;    // Página actual de la paginación
let hasMore = true;     // Si hay más publicaciones por cargar
let isLoading = false;  // Evita peticiones duplicadas

15.2. initFeed()

Se ejecuta 300ms después de cargar la página (espera a que checkSession termine). Según si el usuario está logueado:

  • Logueado: Muestra la caja de nueva publicación con las iniciales del usuario.
  • No logueado: Muestra un prompt invitando a iniciar sesión.

15.3. loadPosts()

Carga las publicaciones desde la API con paginación. Cada vez que se llama, obtiene la siguiente página. Controla el botón "Cargar más" según si hay más resultados.

15.4. renderPost(post)

Genera el HTML de una tarjeta de publicación. Incluye:

  • Header: Avatar con iniciales, nombre del autor, tiempo relativo y badge de tipo de usuario.
  • Contenido: Texto de la publicación e imagen (si existe).
  • Stats: Conteo de likes y comentarios.
  • Acciones: Botones de me gusta, comentar y compartir.
  • Comentarios: Lista de comentarios existentes.
  • Formulario de comentario: Input para agregar un nuevo comentario (oculto por defecto).

15.5. createPost()

Envía la nueva publicación a la API e inserta el resultado al inicio del feed sin recargar la página.

15.6. toggleLike(postId)

Alterna el "me gusta" en una publicación. Actualiza el ícono (🤍 ↔ 💜), la clase CSS y el contador de likes.

15.7. toggleComments(postId) y addComment(postId)

toggleComments muestra/oculta el formulario de comentario. addComment envía el comentario a la API y lo inserta en la lista de comentarios sin recargar.

16. JavaScript — Empleos (empleos.js)

js/empleos.js

Se carga en empleos.html. Gestiona la búsqueda de empleos, postulaciones y publicación de vacantes.

16.1. Sistema de pestañas

switchTab(tab) alterna entre tres vistas: búsqueda de empleos, mis postulaciones y publicar empleo. La pestaña de "publicar" solo es visible para usuarios tipo empresa.

16.2. Búsqueda con debounce

function debounceSearch() {
    clearTimeout(searchTimeout);
    searchTimeout = setTimeout(searchJobs, 300);
}

La técnica de debounce evita hacer una petición por cada tecla que escribe el usuario. Espera 300ms después de la última tecla antes de ejecutar la búsqueda.

16.3. searchJobs()

Envía la búsqueda y filtros a la API y renderiza las tarjetas de empleo. Si no hay resultados, muestra un estado vacío.

16.4. renderJobCard(job)

Genera el HTML de cada tarjeta de empleo con: icono de empresa, tipo de contrato, título, nombre de empresa, descripción, ubicación, fecha, salario y botón de postulación.

16.5. Modal de postulación

openApplyModal() abre un modal con los detalles del empleo y un campo para escribir un mensaje de presentación. submitApplication() envía la postulación a la API.

16.6. loadApplications()

Carga el historial de postulaciones del usuario, mostrando el título del empleo, la empresa y el estado con un badge de color (⏳ Pendiente, ✅ Aceptada, ❌ Rechazada).

16.7. publishJob(e)

Solo para empresas. Envía los datos del formulario de creación de empleo a la API. Después de publicar exitosamente, cambia a la pestaña de búsqueda y recarga los resultados.

17. JavaScript — Donaciones (donaciones.js)

js/donaciones.js

Se carga en donaciones.html. Gestiona el formulario de donación y las estadísticas.

17.1. selectAmount(amount)

Cuando el usuario hace clic en un botón de monto predefinido ($10, $25, $50, etc.), establece ese valor en el input y resalta el botón seleccionado.

17.2. loadDonationStats()

Carga y muestra las estadísticas públicas: total recaudado, número de donaciones, donantes únicos y las 5 donaciones más recientes con avatar, nombre del donante, mensaje y monto.

17.3. handleDonation(e)

Proceso de donación:

  1. Verifica que el usuario esté logueado.
  2. Extrae monto, método de pago y mensaje del formulario.
  3. Valida que el monto sea mayor a 0.
  4. Envía la donación a la API.
  5. Si es exitoso: limpia el formulario y recarga las estadísticas y el historial.

17.4. loadMyDonations()

Carga el historial personal de donaciones del usuario, mostrando monto, método de pago, mensaje y fecha.

js/asesorias.js

18.1. selectType(tipo)

Obtiene la lista de profesionales de la API y renderiza una tarjeta por cada uno con: avatar, nombre, biografía, cantidad de asesorías completadas y botón para solicitar asesoría.

Carga el historial de asesorías. El contenido varía según el rol:

19. Estilos CSS (styles.css)

css/styles.css

Archivo único de estilos que controla toda la apariencia visual de la aplicación. Con más de 50,000 bytes, contiene un diseño moderno y responsivo.

19.1. Variables CSS (Custom Properties)

Define un tema de colores, tamaños de fuente, espaciado y otros valores reutilizables en la raíz del documento (:root):

  • Colores primarios: Tonos púrpura/índigo para la marca.
  • Colores de estado: Verde (éxito), rojo (error), amarillo (advertencia).
  • Escala tipográfica: Desde font-xs hasta font-5xl.
  • Espaciado: Escala predefinida de 4px a 96px.

19.2. Componentes principales estilizados

  • Navbar: Fija en la parte superior, efecto de sombra al hacer scroll, menú responsivo.
  • Hero: Sección de bienvenida con gradiente de fondo.
  • Feature Cards: Tarjetas de características con efecto hover.
  • Post Cards: Tarjetas del feed social con avatar, contenido y acciones.
  • Job Cards: Tarjetas de empleo con diseño grid.
  • Modals: Ventanas modales con overlay oscuro.
  • Toasts: Notificaciones temporales con animación de entrada/salida.
  • Formularios: Inputs, selects y botones con diseño consistente.
  • Badges: Etiquetas de estado con colores semánticos.
  • Animaciones: Fade-in, slide-up y efectos de hover.

19.3. Diseño Responsivo

Usa media queries para adaptar la interfaz a diferentes tamaños de pantalla: escritorio, tablet y móvil. Los layouts se basan en CSS Flexbox y CSS Grid.

20.1. Flujo de registro y login

Usuario visita index.html
    │
    ├─→ Clic "Registrarse"
    │       │
    │       ▼
    │   registro.html → Llena formulario → JS valida datos
    │       │
    │       ▼
    │   auth.js envía POST a api/auth.php?action=register
    │       │
    │       ▼
    │   PHP valida → Encripta contraseña → INSERT en BD → Crea sesión
    │       │
    │       ▼
    │   Redirige a principal.html ✓
    │
    └─→ Clic "Ingresar"
            │
            ▼
        login.html → Email + Password → POST a api/auth.php?action=login
            │
            ▼
        PHP busca usuario → Verifica hash → Crea sesión → Redirige ✓

20.2. Flujo de una publicación

Usuario logueado en principal.html
    │
    ├─→ Escribe texto en la caja de publicación
    │       │
    │       ▼
    │   createPost() → POST a api/publicaciones.php?action=create
    │       │
    │       ▼
    │   PHP inserta en BD → Retorna publicación creada
    │       │
    │       ▼
    │   JS inserta HTML al inicio del feed (sin recargar) ✓
    │
    ├─→ Da "Me gusta" a una publicación
    │       │
    │       ▼
    │   toggleLike() → POST a api/publicaciones.php?action=like
    │       │
    │       ▼
    │   PHP INSERT o DELETE like → Retorna estado y total
    │       │
    │       ▼
    │   JS actualiza ícono y contador ✓
    │
    └─→ Comenta una publicación
            │
            ▼
        addComment() → POST a api/publicaciones.php?action=comment
            │
            ▼
        PHP inserta comentario → Retorna comentario con datos del autor
            │
            ▼
        JS inserta el comentario en el DOM ✓

20.3. Flujo de postulación a empleo

Usuario en empleos.html
    │
    ├─→ Busca empleos (con debounce de 300ms)
    │       │
    │       ▼

            │
            ▼

            │
            ▼
        POST a api/empleos.php?action=apply
            │
            ▼

            │
            ▼
